## セキュリティ

Microsoft は、[Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet)、[Xamarin](https://github.com/xamarin) といった GitHub 組織を含む、当社が管理するすべてのソフトウェア製品とサービスのセキュリティを重視しています。

Microsoft が定義する[セキュリティ脆弱性](https://aka.ms/security.md/definition)に該当する脆弱性を Microsoft 所有のリポジトリで発見したと思われる場合は、以下の手順でご報告ください。

## セキュリティ脆弱性の報告方法

**セキュリティ脆弱性を公開の GitHub Issue で報告しないでください。**

代わりに Microsoft Security Response Center (MSRC) に [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) から報告してください。

ログインせずに送信したい場合は、[secure@microsoft.com](mailto:secure@microsoft.com) へメールしてください。可能であれば、[Microsoft Security Response Center PGP Key](https://aka.ms/security.md/msrc/pgp) から PGP キーを取得して暗号化してください。

通常 24 時間以内に返信します。返答がない場合は、メールでフォローアップして受信状況を確認してください。詳細は [microsoft.com/msrc](https://www.microsoft.com/msrc) を参照してください。

以下の情報を可能な範囲で含めていただくと、調査が迅速になります。

  * 問題の種類（例: バッファオーバーフロー、SQL インジェクション、XSS など）
  * 問題が発生するソースファイルのパス
  * 該当コードの場所（タグ/ブランチ/コミットまたは直接の URL）
  * 再現に必要な特別な設定
  * 再現手順
  * PoC やエクスプロイトコード（可能であれば）
  * 影響範囲と攻撃者が悪用する方法

バグバウンティ対象の場合、詳細な報告はより高額な賞金につながる可能性があります。現在のプログラムは [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) を参照してください。
