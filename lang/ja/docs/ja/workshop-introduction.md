# ワークショップ紹介

## Zava の温度 API を Python から C# へ移行

Zava は Python 実装が中心のサービスを持つ企業を買収しました。
気温・季節・ロケーションのエンドポイントが Zava プロダクトラインで有用です。
Zava のバックエンドと運用ツールは C# ベースのため、
Python サービスを C#/.NET エコシステムへ移行する取り組みを開始しました。

この移行は技術スタック統一、運用簡素化、既存 .NET バックエンドとの統合改善を目的としています。
Python ベースの温度 API を C# へ移行することで、
コンパイル時型安全・実行性能向上・Azure 企業ツールとの整合性が得られます。
API は複数の国/都市/月の履歴天気データを提供し、Zava のスマートファイバー製品に不可欠です。

このワークショップでは、既存の Python API（買収コードベース）を GitHub Copilot で段階的に C# へ移行し、
挙動互換を維持しながら保守性と統合性を高める現実的シナリオを指南します。

### 🎯 なぜ重要か（実運用文脈）

!!! info "本番環境での言語移行を理解する"

    **Zava が C# を選択した理由:**
    
    - 既存 C# バックエンド・内部ツールとの統合が容易になり、統合作業と市場投入を短縮
    - Azure エコシステム（App Insights, Azure AD, IaC 等）によるエンタープライズ展開の強力サポート
    - Visual Studio / Rider / .NET analyzers などエンタープライズ開発ツールで生産性・デバッグ・リファクタリングを向上
    
    **増分検証が本番で重要な理由:**
    
    - **リスク管理**: エンドポイント単位検証で障害範囲を最小化
    - **継続検証**: Python テスト維持で新実装の互換性を保証
    - **信頼構築**: ステップごとの検証が明確なチェックポイントとロールバックポイントを提供
    - **AI から学ぶ**: 小さな変更を検証することで、AI 提案を信頼すべきか見極めやすい

Copilot に対するチャレンジングなリクエストに取り組みながら進めましょう。

    手順は概要を示し、コマンド生成は Copilot を活用してください。

## ワークショップの特徴

Python の HTTP API（買収サービス）を C# (.NET Minimal APIs) に再実装・統合するのが課題です。

### Zava エンタープライズ標準 — 移行の利点

Zava は本番サービスにエンタープライズ標準を適用。C#/.NET への移行は現行 Python 実装より以下で優位:

- セキュリティ/ID: Azure AD, トークン検証, 暗号ライブラリ, セキュリティスキャン/ポリシーツールが成熟
- 可観測性: Application Insights, 構造化ログ (ILogger), 分散トレース, テレメトリ SDK で SLA を満たしやすい
- 信頼性/性能: .NET ランタイム最適化・並行処理・AOT で低遅延高スループット
- 保守性/ガバナンス: 静的型付け, Roslyn アナライザー, NuGet, コード品質チェックで一貫性と監査性を向上
- CI/CD & デプロイ: Azure DevOps / GitHub Actions, コンテナツール, 環境設定パターンで自動化容易
- コンプライアンス/監査: 静的解析, スキャン, バイナリ署名, 依存ポリシー集中管理で準拠
- 開発者生産性/オンボーディング: 豊富な IDE 支援, 決定性ビルド, 高度リファクタリングで短期間に戦力化

Python は多くの領域で優れていますが、Zava にとってエンタープライズ標準達成は C#/.NET が容易です。本ワークショップは挙動を保ちながら段階的に検証する戦略を示します。

以下を行います:

1. Web アプリを起動しブラウザで確認
1. `/docs` エンドポイントでエンドポイント一覧を確認
1. Python の依存・ライブラリは事前インストール済み
1. Python テストで Python/C# 両方の正しさを検証

## GitHub Copilot モード

GitHub Copilot には **Ask**, **Edit**, **Plan**, **Agent** のモードがあります。質問から自律的タスク実行まで段階的に支援します。

### Ask Mode

Q&A アシスタント。コード理解・問題解決・概念学習向け。自然言語で質問し、説明やスニペットを返します。コード変更は行いません。

!!! tip
    クイックな説明やブレインストーミング、サンプル実装に最適。

### Edit Mode

自然言語指示に基づき*直接コード変更*を提案。特定のコードブロックやファイルを指定し、差分で提示するため、最終決定はあなたが行えます。

!!! tip
    リファクタやエラーハンドリング追加などのピンポイント変更向き。複雑作業は Agent Mode 推奨。

### Agent Mode

最も自律的で強力。プロジェクト全体を分析し、タスク計画、編集、コマンド実行、目標達成まで反復します。機能実装・バグ修正・新規コンポーネントスキャフォールディングなど多段タスクに最適。危険な操作は承認を求めるため安全性も確保。

!!! tip
    編集に留まらず、コード/ファイル作成も行います。単発編集より複数ステップが伴う作業に適しています。

### Plan Mode

タスクと目的を明確化し、複雑な作業をステップに分解します。ボイラープレート生成、プロジェクト構造提案、課題事前特定などに役立ちます。

!!! tip
    目標設定やステップ整理で効率を高める際に活用。

## プロンプトでファイル参照を使う

`#file:filename` 記法は重要です:

!!! tip "#file: 参照の使い方"
    **用途:** Copilot に特定ファイルの文脈を渡し、精度を上げる。

    **手順:**
    1. Copilot チャットで `#` を入力
    2. ファイルピッカーから対象ファイルを選択
    3. 残りのプロンプトを入力

    **例:** "add an endpoint" ではなく、`#file:Program.cs add the root endpoint only`

!!! note
    `#file:filename` は、Copilot にコード生成対象ファイルを指示します。`#` を入力しファイルを選択した後、プロンプト本文を貼り付けてください。
