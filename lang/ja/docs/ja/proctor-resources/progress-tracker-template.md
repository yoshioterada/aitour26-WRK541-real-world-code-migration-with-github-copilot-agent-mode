# 参加者進捗トラッカーテンプレート

プロクターがワークショップ中の進捗を把握し、ペース調整やサポート提供の判断をしやすくするためのテンプレートです。

## ワークショップ進捗チェックポイント

各マイルストーンで、どれくらいの参加者が完了したかを確認します:

- 進める / ペースを落とす判断
- サポートが必要な参加者の特定
- 残り時間の調整
- 個別サポートの優先度決定

## ペース判断マトリクス

**80%+ 完了:** ✅ 次へ進む  
**60-79%:** ⚠️ 5-10分延長、ペア支援  
**<60%:** 🛑 グループ解説/デモ

### 80%+ 完了時

- 次へ進む（できた人はチャレンジやヘルプへ）
- 全体に次のステップを簡潔に案内
- 進んでいる参加者にサポートを依頼

### 60-79% 完了時

- 5-10分延長をアナウンス
- 進んでいる参加者とペアリング支援
- プロクターが巡回し課題を特定

### <60% 完了時

- ワークショップを一時停止して全体解説/デモ
- 共通のつまずきを全体で共有し手順を明確化
- タイムラインを調整（コア優先）

## クイックチェック用フレーズ

**環境チェック:**
> 「グリーン=問題なし、イエロー=軽微な問題、レッド=ブロック、で挙手ください」

**理解度チェック:**
> 「わかった方は 👍、不安な方は 👎」

**進捗チェック:**
> 「[checkpoint] ができた方、挙手」

**ペースチェック:**
> 「速い / 遅い / ちょうど良い で教えてください」

**エナジーチェック:**
> 「調子はいかがですか？小休憩が必要ですか？」

## 進捗トラッキングのコツ

1. **頻繁にチェック**（15-20分ごと）
2. **視認性を高める**（ヘルプサインを出しやすく）
3. **ヘルパー活用**（進んでいる参加者にサポートを依頼）
4. **柔軟にペース調整**（リアルタイム情報に基づく）
5. **記録を残す**（次回改善に活用）

**ポイント:** 進捗トラッキングは評価ではなく、全員の成功を支援するためのものです。
